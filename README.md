## Задача 3 
Выберите подходящую систему управления виртуализацией для предложенного сценария. Опишите ваш выбор.
### 3.1
Сценарий: 100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований. Преимущественно Windows based-инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий.
Система: Hyper-V или VMVare
Описание выбора: Hyper-V или VMVare отвечает всем требованиям и позволяет достаточно просто администрировать большие системы.
### 3.2
Сценарий: Требуется наиболее производительное бесплатное open source-решение для виртуализации небольшой (20-30 серверов) инфраструктуры на базе Linux и Windows виртуальных машин.
Система: Xen
Описание выбора: бесплатное source-решение обладающее высокой производительностью.
### 3.3 
Сценарий: Необходимо бесплатное, максимально совместимое и производительное решение для виртуализации Windows-инфраструктуры.
Система: Proxmox 
Описание выбора: бесплатное, максимально совместимое и производительное решение позволяющее развертывать windows сервера.
### 3.4 
Сценарий: Необходимо рабочее окружение для тестирования программного продукта на нескольких дистрибутивах Linux.
Система: KVM
Описание выбора: KVM хорошо подходит для linux и просто в управлении.
## Задача 4
Опишите возможные проблемы и недостатки гетерогенной среды виртуализации (использования нескольких систем управления виртуализацией одновременно) и что необходимо сделать для минимизации этих рисков и проблем. Если бы у вас был выбор, создавали бы вы гетерогенную среду или нет?
### Решение
Проблемы: Гетерогенная среда не позволит свободно и быстро перераспределять виртуальные машины по хостам с разными системами управления виртуализацией в случае выхода из строя одного из хостов. Будет очень сложно настроить централизованое  резервное копирование разных системами управления виртуализацией. Инжинеру, который поддерживает такую инфраструктуру необходимо будет обладать навыками и знаниями разных систем. Если бы у меня был выбор, я бы ни в коем случае не создавал гетерогенную среду виртуализации.
